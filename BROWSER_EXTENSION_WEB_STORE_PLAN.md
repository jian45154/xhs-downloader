# 浏览器扩展：从零实现与 Chrome Web Store 上架路径

## 目标

从零独立实现一个 Chrome 扩展，用于：

1. 在用户主动操作后识别当前小红书笔记中的视频。
2. 下载用户有权使用的视频。
3. 可选将视频和笔记信息交给本机程序处理。
4. 接收本机程序返回的任务状态和转录结果。
5. 通过 Chrome Web Store 审核并公开发布。

原则：不复制现有扩展代码、图标、文案或品牌资产；权限最少；功能单一；不在扩展中执行音频处理或 Whisper 推理。

## 推荐产品边界

首个上架版本只做：

* 识别当前打开的单篇视频笔记。
* 下载一个视频。
* 提取页面标题、正文和视频 URL。
* 检测本机程序是否可用。
* 在用户主动选择后，将任务交给本机程序。
* 展示下载、音频提取和转录状态。

首版不做：

* 批量抓取、账号自动化或绕过访问限制。
* 登录凭据、Cookie 或浏览器存储读取。
* 云端上传、用户账号、支付或订阅。
* 水印去除、版权保护规避或隐藏式下载。
* 在扩展中封装 FFmpeg、Python、Whisper 或模型文件。
* 未经用户操作自动启动下载或转录。

这些限制可以降低版权、隐私、安全和商店审核风险。

## 架构

### 扩展职责

* 使用 Manifest V3。
* Content script 只读取当前页面可见内容和媒体地址。
* Service worker 负责权限请求、媒体下载和任务通信。
* 用户必须点击明确按钮后才开始处理。
* 大文件处理、音频提取和转录全部交给本机程序。

### 与本机程序通信

正式产品优先使用 Chrome Native Messaging：

```text
小红书页面
  ↓
Chrome Extension
  ↓ 视频 URL、标题和任务参数
Native Messaging
  ↓
本机音频转录程序
```

扩展只传递：

* 视频 URL。
* 页面标题和作者名称。
* 用户选择的输出类型。
* 任务 ID。

扩展不通过 Native Messaging 传递完整视频二进制。

如首个原型暂时使用本地 HTTP 服务：

* 服务必须只监听 `127.0.0.1`。
* 必须使用随机安装令牌验证请求。
* 不得只依赖 CORS。
* 正式上架前应决定是否完全迁移到 Native Messaging，避免长期维护两套通信方式。

### 建议权限

使用 Native Messaging 时：

```json
{
  "permissions": [
    "activeTab",
    "downloads",
    "nativeMessaging",
    "scripting",
    "storage"
  ],
  "optional_host_permissions": [
    "https://www.xiaohongshu.com/*",
    "https://*.xhscdn.com/*"
  ]
}
```

尽量在用户首次使用相应功能时请求可选权限，并解释用途。

## 阶段一：版权清理

1. 建立新的空仓库。
2. 不复制当前扩展的 JavaScript、HTML、CSS、图标或文案。
3. 根据公开网页行为重新编写功能。
4. 创建原创名称和视觉资产。
5. 避免名称或图标暗示它是小红书官方产品。
6. 在商店描述中注明与小红书不存在隶属、授权或背书关系。
7. 建立 `THIRD_PARTY_NOTICES.md`，记录所有依赖及许可证。

验收标准：

* 新仓库历史中没有原扩展文件。
* 所有图片和文案均为原创或具有明确商业授权。
* 每个第三方依赖均有可核验许可证。

## 阶段二：实现最小扩展

1. 创建 Manifest V3 基础项目。
2. 只在用户点击扩展按钮后注入解析脚本。
3. 从当前笔记 DOM 中读取：

   * 页面标题或作者名称；
   * 可见正文；
   * 当前视频元素的 `currentSrc`；
   * 必要时读取视频元素内的 `source`。

4. 不扫描浏览历史、其他标签页或后台网络请求。
5. 在下载或发送任务前显示：

   * 即将处理的媒体数量；
   * 是否调用本机程序；
   * 内容权利确认提示。

6. 下载或通信失败时显示真实错误，不得伪报成功。

不要把对页面私有数据结构或混淆脚本的逆向解析作为首版核心方案。DOM 变化时宁可提示暂不支持，也不要扩大抓取范围。

验收标准：

* 只对当前活动标签页生效。
* 没有用户点击时不下载、不发送任务。
* 不读取 Cookie、Local Storage 或账号凭据。
* 图片笔记和无视频页面不会误触发转录。

## 阶段三：接入本机程序

1. 定义稳定的消息协议和版本号。
2. 扩展发送：

   * `task_id`；
   * 视频 URL；
   * 标题和作者；
   * 是否提取音频；
   * 是否生成 TXT、SRT 和 JSON。

3. 本机程序返回：

   * 是否已安装和可用；
   * 当前处理阶段；
   * 进度；
   * 成功输出位置；
   * 可显示给用户的错误。

4. 服务未安装时，扩展只展示官方下载说明，不自动安装程序。
5. 通信中断时允许用户重试，不重复创建任务。
6. 扩展和本机程序必须进行协议版本兼容检查。

验收标准：

* 普通网页不能直接调用本机程序。
* 只有已授权扩展可以创建任务。
* 本机程序关闭或版本不兼容时提示明确。
* 扩展不会因长时间任务导致 Service Worker 状态丢失。

## 阶段四：隐私与安全

编写公开隐私政策，至少说明：

* 扩展读取哪些页面信息。
* 视频 URL、标题和正文为何被处理。
* 哪些信息会发送给本机程序。
* 音频和转录默认只在用户电脑本地处理。
* 是否存储任何数据以及保存期限。
* 不出售或共享用户数据。
* 用户如何删除本地数据并卸载扩展。
* 联系方式和政策生效日期。

安全措施：

* 所有页面数据都视为不可信。
* 文件名和任务参数必须校验。
* 不执行下载内容。
* 不使用远程托管代码、`eval` 或动态脚本。
* 不在扩展源码中保存密钥或固定认证令牌。
* 固定依赖版本并记录软件物料。
* 没有遥测，或遥测为明确选择加入。

验收标准：

* Chrome 扩展源码中没有密钥。
* 隐私政策与实际行为逐项一致。
* 删除扩展后不保留浏览和任务数据。
* 权限均能对应到明确的用户功能。

## 阶段五：商店材料

准备：

* 扩展名称、简短描述和详细描述。
* 原创的 128×128 图标。
* 至少一组真实功能截图。
* 隐私政策公开 URL。
* 支持邮箱和支持页面。
* 单一用途说明。
* 每项权限的必要性解释。
* Native Messaging 用途说明。
* 数据使用披露。
* 内容权利与非官方声明。

建议商店文案避免：

* “无限下载”“破解”“去限制”等表述。
* 宣称支持下载任何受版权保护内容。
* 暗示获得小红书官方授权。
* 无法证明的隐私、安全或性能承诺。

建议加入：

> 本工具仅用于保存你本人创作、已获授权或法律允许使用的内容。请遵守平台条款和适用法律。本产品与小红书无隶属或授权关系。

## 阶段六：审核前测试

测试矩阵：

* Windows 10 和 Windows 11。
* Chrome 当前稳定版。
* 本机程序已安装、未安装、未启动和版本不兼容。
* 图文笔记、普通视频笔记、长视频和无声视频。
* CDN 请求失败、页面结构变化和超大文件。
* 非小红书页面。
* 扩展更新期间的任务恢复。

发布包检查：

* 删除日志、缓存、模型文件、测试视频和源码映射。
* ZIP 根目录直接包含 `manifest.json`。
* 不包含 FFmpeg、Python、Whisper 或可执行文件。
* 不包含开发服务器地址和调试开关。
* 使用全新 Chrome Profile 完成安装测试。

## 阶段七：提交与发布

1. 注册 Chrome Web Store 开发者账户并启用两步验证。
2. 支付一次性开发者注册费。
3. 上传扩展 ZIP。
4. 完成隐私、权限和数据使用表单。
5. 提交为非公开测试版本。
6. 用新用户环境完成安装与卸载测试。
7. 修正审核反馈。
8. 审核稳定后改为公开发布。

本机程序应通过独立官网或可信发布渠道提供，不得将可执行文件放入扩展包。

## 收费策略

首次上架建议免费，先验证审核稳定性和真实需求。首版不同时引入支付、账号和云服务。

后续低风险收费路径：

1. 扩展保留免费的基础识别和下载功能。
2. 本机程序提供付费高级功能。
3. 使用外部网站完成购买和许可证管理。
4. 在扩展中清楚披露哪些功能需要本机程序或付费许可。
5. 不将浏览数据用于广告或信用评估。

## 上架门槛

只有满足以下条件才提交审核：

* [ ] 代码和视觉资产均为独立实现。
* [ ] 第三方许可证完整。
* [ ] 权限已压缩到最小范围。
* [ ] 本机通信具备明确认证和版本控制。
* [ ] 隐私政策与实际行为一致。
* [ ] 不读取账号凭据或绕过访问控制。
* [ ] 版权提示和非官方声明清楚可见。
* [ ] 全新 Chrome Profile 测试通过。
* [ ] 发布包不含日志、缓存、模型和可执行文件。
* [ ] 支持、卸载和数据删除流程可用。

## 推荐执行顺序

1. 新建干净的扩展仓库。
2. 完成独立 UI 和原创图标。
3. 实现只处理当前页面的单视频识别与下载。
4. 定义与本机程序的消息协议。
5. 接入本机程序状态和任务进度。
6. 编写隐私政策、许可证和支持文档。
7. 完成安全测试和发布包清理。
8. 先以免费、非公开版本提交审核。
9. 审核稳定后公开发布，再评估收费。
